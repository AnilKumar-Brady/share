Authentication

Login

/api/auth/login

The Login API allows users to authenticate themselves by providing their username and password. Upon successful authentication, a JSON Web Token (JWT) is returned, which can be used to authorize subsequent API requests.

Request Body schema: application/json

Model

userName
string

The username of the user.

password
string

The password of the user.

Responses

Request samples

  • Payload
Content type
application/json
{
  • "userName": "string",
  • "password": "string"
}

Refresh Token

/api/auth/refreshtoken

The Refresh Token API allows users to obtain a new access token by providing a valid refresh token.

Request Body schema: application/json

Model

token
string

Access token.

refreshToken
string

The refresh token used for refreshing or revoking the access token.

Responses

Request samples

  • Payload
Content type
application/json
{
  • "token": "string",
  • "refreshToken": "string"
}

Revoke Token

/api/auth/revoketoken

The Revoke Token API allows users to revoke (invalidate) an access or refresh token, effectively rendering it unusable for future authentication or authorization.

Request Body schema: application/json
token
string

Access token.

refreshToken
string

The refresh token used for refreshing or revoking the access token.

Responses

Request samples

  • Payload
Content type
application/json
{
  • "token": "string",
  • "refreshToken": "string"
}

Change Password

/api/auth/ChangePassword

The Change Password API allows users to update their password by providing their current password and specifying a new password.

Request Body schema: application/json
